Level 1 & 2: Syntax and Type SystemsSyntax vs. Logical Errors: A syntax error (like a missing colon in Python) prevents the program from starting because the interpreter doesn't understand the "grammar". A logical error (like using > instead of <) allows the code to run, but it gives the wrong result, making it harder to find.Static vs. Dynamic Typing: Static typing (Java) is safer for SACCO financial systems because it catches type errors (like putting text in a money field) before the code ever runs. Dynamic typing (Python) is better for Kenyan startups because it allows for faster coding and flexibility.

Level 3: Control Flow (Machakos Advisory)
If-Else vs. Switch: For the rainfall rules, if-else is better because we are checking "ranges" (e.g., rainfall < 200). A switch statement is usually better for specific, single values (like picking a menu option 1, 2, or 3).

Level 4: Abstraction (NHIF System)
Maintainability: Using a Patient class (OOP) is more organized. If NHIF changes the 10% co-payment rule to 15%, I only change one line in the class method instead of searching through the entire program to find every calculation.

Level 5: Paradigm Comparison (Ride-Hailing)
Extension: The Object-Oriented version was the easiest to extend.
Scalability: If we added surge pricing for Nairobi traffic, OOP scales best because we can add a "Surge" property to the Trip object without rewriting the basic pricing logic

Level 6: Scope and Error Handling
Security: Lexical scope improves security by ensuring sensitive patient data is only visible inside the specific function that needs it.
Risk: Dynamic scope is risky in hospitals because a variable's value could change depending on which function was called last, potentially leading to unauthorized access to records.

Level 7: Capstone (Mobile Money DSL)
Prototyping: Python is ideal for DSL prototyping because its string handling (like .split()) is very simple. In C++, parsing a command like "TRANSFER..." would require much more complex code.

AI in Kenya: AI assistants help by generating the logic to parse complex transaction strings quickly, helping Kenyan developers build fintech tools faster.
